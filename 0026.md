## 0. 원칙

1. **하나의 계정, 두 얼굴** — 개인 얼굴(닉네임·펫·개인 글)과 업체 얼굴(상호·사업장·업체 글)은 같은 계정 위의 별개 정체성이다.
2. **진입 맥락이 얼굴을 결정한다** — 보는 사람이 어느 문으로 들어왔는지가 어떤 얼굴을 보는지를 정하며, 계정 주인의 현재 모드는 "지금 어떤 얼굴로 활동 중인가(작성 태그·검색 노출·내 화면)"만 정한다.
3. **연결 비노출** — 어떤 사용자가 어떤 업체를 운영하는지는 어떤 경로로도 드러나지 않아야 한다. 두 얼굴을 잇는 표시(배지·닉네임·동네)는 서로의 얼굴에 나타나지 않는다.

## 1. 데이터 축

| 축 | 위치 | 의미 |
| --- | --- | --- |
| `users.active_mode` | personal/business | 주인의 현재 활동 모드 — 작성 태그의 원천, 검색 노출 정체성, 내 화면 얼굴 |
| `posts.authored_as` · `comments.authored_as` | 행 스냅샷(INSERT 트리거) | 글·댓글이 어느 얼굴로 쓰였는지 — 모드를 바꿔도 과거 작성물의 얼굴은 불변 |

| `business_profiles.photo_*` → `facilities.owner_*` | 원본→지도 사본 | 대표 사진·간판명·전화의 지도 동기화(`owner_updated_at` 으로 월 재적재에서 보존) |

## 2. 노출 규칙 (서버)

- **public_profiles**(07-15 개정): 업체 필드(`business_name/category/address/phone/facility_id`)는 **승인(approved)이면 상시 공개** — 주인 모드와 무관. '일반 모드 = 영업 중단'이 되어 상호 검색·업체 프로필이 사라지던 결합을 해제했다. active_mode 는 작성 태그·내 화면 얼굴·매칭 차단만 담당.
- **검색**: 얼굴별 2쿼리 — 닉네임 매칭은 개인 얼굴 결과(업체 정보 미포함), 상호 매칭은 업체 얼굴 결과. 같은 계정이 두 얼굴로 각각 검색되며 서로 연결은 UI 에 노출되지 않는다. (종전의 '업체 모드 중 닉네임 검색 불가' 트레이드오프 해소.)
- **v_post_feed**: `authored_as='business'` 글은 작성자 = **상호**(승인 업체, 모드 무관), 작성자 동네(author_address) **비노출**. `authored_as` 를 노출해 앱이 작성자 탭 얼굴을 라우팅.
- **v_comment_feed**: 업체 모드로 단 댓글도 작성자 = **상호**(동일 규칙, `comments.authored_as` 스냅샷). 댓글은 작성자 탭이 없어 표시 치환만으로 완결.
- **v_chat_rooms**: 업체 문의 방의 상대는 **상호로 표시**(business_profiles 직접 조인 — 주인이 개인 모드로 돌아가도 유지) + `context` 노출.
- **start_direct_chat(p_other, p_context)**: 업체 문의는 상대가 승인 업체일 때만. canonical_key 에 `:biz:<업체측>` 포함.

## 3. 얼굴 라우팅 (앱)

`UserProfileScreen.forcePersonalFace` — 개인 맥락 진입은 상대가 업체 모드여도 개인 얼굴 고정:

| 진입점 | 얼굴 |
| --- | --- |
| 상호 검색 결과 / 업체 글 작성자 / 업체 문의 방 헤더 | 업체 |
| 닉네임 검색 / 팔로우(Pawing·Pawmate) 목록 / 펫 보호자 목록 / 개인 채팅방 / 개인 글 작성자 | 개인(강제) |

업체 얼굴 화면: 상호가 이름 자리(수축 바 포함), 섹션 = 업체 정보(업종·주소·전화걸기·인증 배지) / 방문 후기(매칭 시설) / 업체 게시글. 개인 얼굴: 기존 그대로(펫·받은 평가·개인 글), 업체 흔적 없음.

## 6. 사용성 트레이드오프 (인지하고 수용한 것)

- **API 수준 연결 노출 한계** ⚠️ — public_profiles 행에 닉네임과 업체 필드가 공존하므로, 앱 UI 는 얼굴을 분리하지만 anon 키로 REST 를 직접 조회하면 닉네임↔상호 연결을 볼 수 있다. 완전 차단은 업체 얼굴에 불투명 식별자(별도 business_id)를 부여하는 리팩터가 필요 — 일반 사용자 위협 모델에선 현 수준 수용, 강화 필요 시 후속.
- 업체 모드 중 팔로워가 프로필을 열면 개인 얼굴이 보인다(개인 맥락) — 주인이 "지금 업체로 활동 중"임은 드러나지 않는다.
