# 0021 — 주변 반려동물 시설 지도 (공공데이터 적재 → PostGIS 반경조회 → 네이버 지도 마커)

> PawMate(`pmdart`) + Supabase 백엔드(`pmdb`) 기준.
> **선행 의존: 0017(지역 인증 — geolocator/네이버 지도/region_code).**
> 지도 SDK: **flutter_naver_map**(0017에서 도입). 위치: **geolocator**(0017).

---

## 0. 목표와 결정 요약

- 지도에 **반려동물 관련 시설**(동물병원·애견미용·반려동물호텔·애견카페)만 골라 마커로 표시한다.
- **데이터 소스가 두 갈래(하이브리드)다:**
  - **동물병원·애견미용·반려동물호텔 → 공공데이터 CSV 를 DB 에 적재**(LOCALDATA / data.go.kr). 업체명·주소·전화번호·영업상태가 공개돼 있다. 이 3종은 **DB 조회**만 한다.
  - **애견카페 → 네이버 지역검색 API 를 실시간 호출**. 공공데이터에 애견카페 전용 업종이 없어 별도 처리. DB 에 적재하지 않고 지도 진입 시 현재 위치로 검색.
- **CSV 3종은 실시간 외부 호출을 하지 않는다.** 흐름은 **공공데이터 CSV → Supabase DB 적재 → Flutter 가 DB 조회 → 네이버 지도 마커**. 애견카페만 예외적으로 실시간이다.
- 앱 실행 시 **현재 위치 기준 반경 5km** 시설을 Supabase 에서 조회해 지도에 표시한다.
- 각 지역에 올라온 **게시글**도 지도에 모아 표시한다 — **행정동 단위 클러스터 마커**(예: "역삼동 12").

### 결정값(확정)

| 항목 | 결정 | 비고 |
| --- | --- | --- |
| 조회 반경 | **고정 5km** | 도심 마커 과밀은 클러스터링으로 완화 |
| 공간 조회 | **PostGIS** `geography` + GIST + `ST_DWithin` | RPC 로 노출 (CSV 3종) |
| 데이터 소스 | **병원·미용·호텔 = CSV→DB / 애견카페 = 네이버 지역검색 실시간** | 하이브리드 |
| CSV 갱신 | **주기적 자동**(Edge Function + cron, 예: 월 1회) | 영업상태·신규 업체 반영 |
| 애견카페 | **실시간 네이버 지역검색 API**(Edge Function 프록시) | DB 미적재, 키 서버 보관 |
| 애견카페 커버리지 | **v1 단일 호출(상위 5곳)** → 밀집지 불만 시 격자 승급 | 호출부만 교체, 앱/스키마 무변경 |
| 게시글 표시 | **행정동 클러스터 마커** | 0017 `region_code` 와 정합, 위치 노출 적음 |
| 좌표 처리 | **적재 시 EPSG:5174 → 4326 변환** | LOCALDATA 는 위경도 미제공(중부원점TM) |

### ★ 좌표계 주의 (가장 중요한 함정)

LOCALDATA 동물 인허가 데이터의 좌표는 **위경도(WGS84)가 아니라 보정 전 Bessel 중부원점TM(EPSG:5174)** 으로 제공된다(위경도는 제공 안 함). 그대로 지도에 찍으면 엉뚱한 위치가 된다. 따라서 **적재 단계에서 5174 → 4326(WGS84)로 변환**해 `geography(Point,4326)` 로 저장한다. 변환은 PostGIS `ST_Transform` 으로 DB 안에서 처리한다(아래 §3).

## 1. 전체 플로우

```
═══ 경로 A: 병원·미용·호텔 (CSV → DB, 주기 적재) ═══
[공공데이터포털/LOCALDATA]  (동물병원·미용·위탁관리 인허가 CSV)
   │  ① sync-facilities 가 CSV 수집
   ▼
[Edge Function: sync-facilities]  (service_role, cron 주기 실행)
   │  ② 파싱 → 카테고리 매핑 → upsert(ext_id 기준)
   │  ③ 좌표 5174 → 4326 변환(ST_Transform) → geom 저장
   ▼
[Supabase: public.facilities]  (PostGIS, GIST 인덱스)
   ▲  ④ 앱 진입 시 현재 좌표 + 반경 5km RPC (실시간 외부 호출 X)
   │     app.facilities_within(lng,lat,5000,categories[])
   │
═══ 경로 B: 애견카페 (네이버 실시간) ═══
[Edge Function: search-petcafe]  ──→ [네이버 지역검색 API]
   ▲  ④' 앱 진입 시 현재 좌표로 실시간 검색(키 서버 보관, KATECH→WGS84, 5건)
   │
[pmdart: 지도 화면]  (A 결과 + B 결과 합쳐 마커)
   │  ⑤ 시설 마커(카테고리별 아이콘) + 게시글 행정동 클러스터 마커
   ▼
[네이버 지도]  마커/클러스터 렌더링
```

게시글 클러스터는 별도 경로다: `app.posts_by_region(bbox)` → 행정동별 개수 + 대표 좌표 → 동 위에 "n" 마커.

## 2. 스키마 — `facilities` (PostGIS)

```sql
-- migration: 2026XXXX_facilities.sql
create extension if not exists postgis;   -- Supabase 기본 지원

-- 카테고리(소비자에게 보일 반려동물 시설만)
-- ※ facilities 테이블에 적재되는 건 CSV 3종(병원·미용·호텔)뿐.
--   pet_cafe 는 §4-1 네이버 실시간이라 DB 미적재(마커 표시용 라벨로만 사용).
--   pet_sales 는 현재 범위 밖(추후 CSV 보강 시 추가) — 미사용.
create type public.facility_category as enum (
  'animal_hospital',   -- 동물병원        (CSV 적재)
  'grooming',          -- 동물미용업       (CSV 적재)
  'pet_hotel',         -- 동물위탁관리업(호텔·유치원) (CSV 적재)
  'pet_cafe',          -- 애견카페         (네이버 실시간, DB 미적재)
  'pet_sales'          -- 동물판매업       (현재 미사용, 추후 보강)
);

create table public.facilities (
  id            uuid primary key default gen_random_uuid(),
  category      public.facility_category not null,
  source        varchar(30) not null,            -- 'localdata' | 'data_go_kr'
  ext_id        varchar(120) not null,           -- 원천 관리번호(중복 방지 키)
  name          varchar(200) not null,           -- 사업장명
  address       text,                            -- 소재지(도로명/지번)
  phone         varchar(40),
  biz_status    varchar(20),                     -- '영업/정상' | '폐업' 등 원천 상태
  is_open       boolean not null default true,   -- 영업중만 지도 노출(파생)
  license_date  date,                            -- 인허가일자
  region_code   varchar(20),                     -- 행정동코드(역지오코딩, 선택; posts.region_code 와 동형)
  -- 원천 좌표(검증·재변환 대비 보존)
  src_x         double precision,                -- EPSG:5174 X
  src_y         double precision,                -- EPSG:5174 Y
  geom          geography(Point, 4326),          -- 변환 결과(WGS84)
  created_at    timestamptz not null default now(),
  updated_at    timestamptz not null default now(),
  constraint facilities_src_uq unique (source, ext_id)
);

-- ★ 반경 조회 핵심: 공간 인덱스
create index facilities_geom_gix on public.facilities using gist (geom);
create index facilities_cat_idx  on public.facilities (category) where is_open;

comment on table public.facilities is
  '공공데이터 반려동물 시설. 주기 동기화. geom 은 5174→4326 변환 저장.';
```

> **RLS/GRANT:** 시설은 공개 정보이므로 `authenticated`(필요 시 `anon`)에게 **SELECT 만** 허용한다. INSERT/UPDATE/DELETE 는 **service_role(적재 함수)만**. 신규 테이블이라 0005 기본권한을 상속하지 않으므로 GRANT 를 명시한다.

```sql
alter table public.facilities enable row level security;
grant select on public.facilities to authenticated;     -- (공개로 하려면 anon 도)
create policy facilities_select_all on public.facilities
  for select using (true);                                -- 영업/폐업 필터는 RPC/쿼리에서
-- INSERT/UPDATE/DELETE GRANT 없음 → 클라이언트 쓰기 불가(service_role 만)
```

## 3. 반경 조회 RPC (PostGIS)

클라이언트는 좌표·반경·카테고리만 넘기고, DB 가 공간 인덱스로 5km 내 시설을 돌려준다.

```sql
-- migration: 2026XXXX_facilities_within_rpc.sql
create or replace function app.facilities_within(
  p_lng        double precision,
  p_lat        double precision,
  p_radius_m   integer default 5000,             -- 고정 5km(필요 시 상한 검증)
  p_categories public.facility_category[] default null
) returns table (
  id uuid, category public.facility_category, name varchar, address text,
  phone varchar, is_open boolean, lng double precision, lat double precision,
  distance_m double precision
)
language sql stable
as $$
  select f.id, f.category, f.name, f.address, f.phone, f.is_open,
         st_x(f.geom::geometry) as lng, st_y(f.geom::geometry) as lat,
         st_distance(f.geom, st_makepoint(p_lng, p_lat)::geography) as distance_m
    from public.facilities f
   where f.is_open
     and (p_categories is null or f.category = any(p_categories))
     and st_dwithin(f.geom, st_makepoint(p_lng, p_lat)::geography,
                    least(p_radius_m, 5000))     -- 서버에서 5km 상한 강제
   order by distance_m
   limit 500;                                     -- 과밀 방어
$$;
grant execute on function app.facilities_within(double precision,double precision,integer,public.facility_category[])
  to authenticated;
```

- `st_dwithin(geography, geography, m)` 는 **미터 단위**라 좌표계 환산 불필요.
- GIST 인덱스 덕에 5km 반경도 빠르다. `limit 500` 으로 도심 과밀 시 응답 크기 제한.
- `least(p_radius_m, 5000)` 로 **클라가 더 큰 반경을 보내도 5km 로 잘라** 부하를 막는다.

## 4. 적재 Edge Function `sync-facilities` (CSV 3종 주기 동기화)

> **이 절은 동물병원·애견미용·반려동물호텔만 다룬다.** 애견카페는 적재하지 않고 §4-1 실시간 네이버 검색으로 처리한다.

위치: `supabase/functions/sync-facilities/index.ts`. **service_role** 로 동작, 사용자 JWT 불필요(cron 트리거). 외부에 노출하지 않거나 별도 시크릿 헤더로 보호.

처리:

```ts
// 1) 공공데이터 CSV 수집 (카테고리별 데이터셋)
//    LOCALDATA/data.go.kr CSV 다운로드 → 파싱. (병원·미용·호텔만)
//    응답 필드 예: 사업장명, 소재지전체주소, 전화번호, 영업상태명,
//                  인허가일자, 좌표정보(x,y; EPSG:5174), 관리번호(ext_id)
const rows = await fetchPublicCsv(category);   // 카테고리→CSV 데이터셋 매핑표 사용

// 2) 정규화 → upsert 페이로드
const payload = rows.map(r => ({
  category, source: "localdata", ext_id: r.관리번호,
  name: r.사업장명, address: r.소재지전체주소, phone: r.소재지전화 ?? null,
  biz_status: r.영업상태명, is_open: /영업|정상/.test(r.영업상태명 ?? ""),
  license_date: parseDate(r.인허가일자),
  src_x: num(r.좌표정보_x), src_y: num(r.좌표정보_y),
}));

// 3) DB upsert + 좌표변환은 DB 안에서(ST_Transform). RPC 로 한 번에 처리.
await admin.rpc("upsert_facilities", { p_rows: payload });
```

좌표 변환을 **DB 에서** 처리하는 upsert RPC:

```sql
-- migration: 2026XXXX_upsert_facilities_rpc.sql
create or replace function app.upsert_facilities(p_rows jsonb)
returns integer
language plpgsql security definer set search_path = ''
as $$
declare n integer;
begin
  insert into public.facilities as f
    (category, source, ext_id, name, address, phone, biz_status, is_open,
     license_date, src_x, src_y, geom, updated_at)
  select (e->>'category')::public.facility_category, e->>'source', e->>'ext_id',
         e->>'name', e->>'address', e->>'phone', e->>'biz_status',
         coalesce((e->>'is_open')::boolean, true), (e->>'license_date')::date,
         (e->>'src_x')::double precision, (e->>'src_y')::double precision,
         -- ★ 5174 → 4326 변환. 좌표 없으면 null.
         case when e->>'src_x' is not null and e->>'src_y' is not null then
           st_transform(
             st_setsrid(st_makepoint((e->>'src_x')::double precision,
                                     (e->>'src_y')::double precision), 5174),
             4326)::geography
         end,
         now()
    from jsonb_array_elements(p_rows) as e
  on conflict (source, ext_id) do update
    set name = excluded.name, address = excluded.address, phone = excluded.phone,
        biz_status = excluded.biz_status, is_open = excluded.is_open,
        license_date = excluded.license_date,
        src_x = excluded.src_x, src_y = excluded.src_y, geom = excluded.geom,
        updated_at = now();
  get diagnostics n = row_count;
  return n;
end;
$$;
revoke all on function app.upsert_facilities(jsonb) from public, anon, authenticated;
```

cron 등록(pg_cron + pg_net 으로 Edge Function 호출, 또는 플랫폼 스케줄):

```sql
-- 예: 매월 1일 03:00(KST) 동기화
select cron.schedule('sync-facilities-monthly', '0 18 1 * *',  -- UTC 기준(=KST 03:00)
  $$ select net.http_post(
       url    := 'https://<ref>.functions.supabase.co/sync-facilities',
       headers:= jsonb_build_object('x-sync-secret', '<SYNC_SECRET>')
     ); $$);
```

> **EPSG:5174 검증:** PostGIS 의 5174 정의(보정 전 Bessel 중부원점)와 LOCALDATA 좌표가 맞는지, **위치를 아는 업체 몇 곳**(예: 유명 동물병원)으로 변환 결과를 지도에 찍어 ±수 m 오차인지 확인한다. 만약 어긋나면 5174 대신 보정계수가 들어간 정의(예: 5179 계열) 또는 사용자정의 SRID 를 검토한다(데이터셋 안내 문구 확인).

## 4-1. 애견카페 — 네이버 지역검색 실시간 (Edge Function 프록시)

애견카페는 공공데이터에 전용 업종이 없어 **네이버 지역검색 API**(`openapi.naver.com/v1/search/local.json`)로 실시간 검색한다. DB 에 적재하지 않는다.

**왜 Flutter 가 네이버를 직접 부르지 않고 Edge Function 프록시를 두나:**

- **키 보호:** 네이버 `X-Naver-Client-Id/Secret` 을 앱에 넣으면 추출당해 도용된다. 키는 **Edge Function secret** 에만 둔다.
- **호출 통제:** 남용·쿼터 초과를 서버에서 캐시·레이트리밋으로 막을 수 있다(동일 지역 결과를 짧게 캐싱).

```ts
// supabase/functions/search-petcafe/index.ts  (로그인 JWT 검증, --no-verify-jwt)
// 입력: { lat, lng }  →  현재 위치 기준 "애견카페" 검색
const q = encodeURIComponent("애견카페");
const res = await fetch(
  `https://openapi.naver.com/v1/search/local.json?query=${q}&display=5&sort=comment`,
  { headers: {
      "X-Naver-Client-Id": Deno.env.get("NAVER_CLIENT_ID")!,
      "X-Naver-Client-Secret": Deno.env.get("NAVER_CLIENT_SECRET")!,
  }});
const j = await res.json();
// 좌표: 네이버 지역검색은 mapx/mapy 를 KATECH(TM128) 정수로 준다.
//       → WGS84 로 변환해야 지도에 맞는다(아래 ★).
const items = j.items.map((it: any) => ({
  name: it.title.replace(/<[^>]+>/g, ""),   // <b> 태그 제거
  address: it.roadAddress ?? it.address,
  category: "pet_cafe",
  ...katechToWgs84(Number(it.mapx), Number(it.mapy)),  // → { lat, lng }
}));
return Response.json({ items });
```

**★ 두 가지 큰 제약 (반드시 인지):**

1. **결과 최대 5건** — 네이버 지역검색은 `display` 최댓값이 5다. "반경 5km 애견카페 전부"가 아니라 **검색어 관련 상위 5곳**만 온다.
2. **좌표계** — 지역검색의 `mapx/mapy` 는 **KATECH(TM128)** 정수다(위경도 아님). `katechToWgs84` 변환 필수. CSV(EPSG:5174)와는 **다른 좌표계**이니 변환 로직을 혼동하지 말 것.

> 검색이 "현재 위치 반경"이 아니라 "검색어 매칭"이라, 위치 정렬이 약하다. 받은 5건을 클라에서 현재 좌표 기준 거리 계산해 5km 밖이면 숨기는 후필터를 두는 게 좋다.

### 커버리지 전략 — 단일 호출로 시작(확정), 격자는 향후 승급

| 단계 | 방식 | 특성 |
| --- | --- | --- |
| **v1 (확정)** | **단일 호출(상위 5곳)** | 호출 1번, 단순·쿼터 거의 안 씀. 밀집 지역(강남·홍대)에선 누락 가능 |
| v2 (승급) | 격자 다중질의 | 영역을 N칸으로 쪼개 칸당 검색 → 합치고 **중복 제거**. 거의 전수, 호출·복잡도 N배 |

- 우리나라 대부분 동네는 애견카페가 드물어 **상위 5곳으로 체감상 충분**하다. 그래서 v1 으로 출발.
- **승급 트리거:** 밀집 지역에서 "근처 카페가 안 뜬다"는 피드백/지표가 보이면 그때 격자로 전환. 코드상 `search-petcafe` 의 검색 호출부만 `단일 → 격자 루프(+dedup)`로 교체하면 되도록, 호출 로직을 함수로 분리해 둔다(앱/스키마 변경 없음).
- v1 에서도 받은 5건은 **5km 후필터**로 위치 정렬 약점을 보완한다.

## 5. Flutter — 지도 화면 (pmdart)

```dart
// 1) 앱 실행/지도 진입 시 현재 위치 (0017 LocationService 재사용)
final pos = await LocationService.current();          // lat/lng/accuracy

// 2a) 병원·미용·호텔: 반경 5km DB 조회 (실시간 외부 호출 아님)
final rows = await supabase.rpc('facilities_within', params: {
  'p_lng': pos.lng, 'p_lat': pos.lat, 'p_radius_m': 5000,
  'p_categories': dbCategories,                       // 애견카페 제외한 3종
});

// 2b) 애견카페: 실시간 네이버 지역검색 (Edge Function 프록시)
List petCafes = [];
if (showPetCafe) {
  final r = await supabase.functions.invoke('search-petcafe',
      body: {'lat': pos.lat, 'lng': pos.lng});
  petCafes = (r.data['items'] as List)
      .where((c) => distMeters(pos, c) <= 5000)        // 5km 후필터(§4-1 참고)
      .toList();
}
final allFacilities = [...rows, ...petCafes];

// 3) 네이버 지도 마커 (flutter_naver_map)
for (final f in allFacilities) {
  final marker = NMarker(
    id: 'fac_${f['id']}',
    position: NLatLng(f['lat'], f['lng']),
    icon: _iconFor(f['category']),                    // 카테고리별 아이콘
  );
  marker.setOnTapListener((_) => _showFacilitySheet(f)); // 이름·주소·전화·영업상태
  controller.addOverlay(marker);
}
```

- **카테고리 필터 토글**(병원/미용/카페/호텔/판매)로 `selectedCategories` 만 바꿔 재조회.
- 마커 탭 → 바텀시트에 업체명·주소·전화(전화 걸기)·영업상태·거리(`distance_m`).
- 위치 권한 거부 시 0017 의 처리(기본 지역/안내) 재사용.
- **재조회 트리거:** 지도 이동이 크면(중심 좌표가 일정 거리 이상 벗어나면) 새 중심으로 다시 RPC. 매 프레임이 아니라 디바운스.

패키지: `flutter_naver_map`, `geolocator`(둘 다 0017에서 도입), `supabase_flutter`.

## 6. 게시글 — 행정동 클러스터 마커

게시글은 개별 위치 핀이 아니라 **행정동 단위로 묶어** 표시한다(개인 위치 노출 최소화, 0017 `region_code` 와 정합).

```sql
-- migration: 2026XXXX_posts_by_region_rpc.sql
-- 화면에 보이는 영역(bbox) 안의 행정동별 게시글 수 + 대표 좌표
create or replace function app.posts_by_region(
  p_min_lng double precision, p_min_lat double precision,
  p_max_lng double precision, p_max_lat double precision
) returns table (region_code varchar, post_count bigint,
                 lng double precision, lat double precision)
language sql stable
as $$
  select d.region_code, count(p.id) as post_count, d.lng, d.lat
    from public.dong_centroids d
    join public.posts p on p.region_code = d.region_code
   where d.lng between p_min_lng and p_max_lng
     and d.lat between p_min_lat and p_max_lat
     and p.visibility_status = 'visible'      -- hidden/deleted 제외(deleted_at 도 자동 null)
   group by d.region_code, d.lng, d.lat
   having count(p.id) > 0;
$$;
grant execute on function app.posts_by_region(double precision,double precision,double precision,double precision)
  to authenticated;
```

대표 좌표(행정동 중심점)가 필요하다. 두 가지 방법 중 택1(서브 결정):

- **(권장) `dong_centroids` 테이블**: 행정동 경계 공공데이터(행정안전부 행정동 경계)에서 동별 중심점(centroid)을 미리 계산해 `region_code → (lng,lat)` 로 저장. 정확하고 가볍다.
- **게시글 좌표 평균**: `posts.display_lat/display_lng`(공개용 오프셋 좌표, 0002 스키마)가 있는 글만 동별로 평균. 별도 테이블 없이 가능하지만 좌표 없는 글은 누락 → 부정확. `dong_centroids` 없을 때 보조용.

```dart
// Flutter: 보이는 영역 bbox 로 조회 → 동마다 클러스터 마커
final regions = await supabase.rpc('posts_by_region', params: {
  'p_min_lng': bounds.southWest.longitude, 'p_min_lat': bounds.southWest.latitude,
  'p_max_lng': bounds.northEast.longitude, 'p_max_lat': bounds.northEast.latitude,
});
for (final r in regions) {
  final m = NMarker(id: 'dong_${r['region_code']}',
    position: NLatLng(r['lat'], r['lng']),
    icon: _badgeIcon('${r['post_count']}'));          // 숫자 배지
  m.setOnTapListener((_) => _openRegionPosts(r['region_code'])); // 그 동 글 목록
  controller.addOverlay(m);
}
```

> 시설 마커와 게시글 클러스터는 **별 레이어**로 두고, 상단 토글로 "시설 / 게시글"을 켜고 끌 수 있게 하면 지도가 깔끔하다(시설 과밀 시 특히).

## 7. 보안 / 정책

| 항목 | 처리 |
| --- | --- |
| 시설 데이터 쓰기 | service_role(적재 함수)만. 클라는 SELECT 전용(GRANT 로 제한) |
| 적재 함수 무단 호출 | cron 전용 + `x-sync-secret` 헤더 검증, 공개 URL 미노출 |
| 네이버 키 보호(애견카페) | `NAVER_CLIENT_ID/SECRET` 은 Edge Function secret 에만. 앱에 미포함 |
| 애견카페 실시간 남용 | `search-petcafe` 는 로그인 JWT 검증 + 서버 캐시·레이트리밋(쿼터 보호) |
| 반경 남용(거대 반경) | RPC 에서 `least(p_radius_m, 5000)` 상한 + `limit 500` |
| 개인 위치 노출(게시글) | 개별 핀 대신 **행정동 클러스터**(좌표를 동 중심으로 일반화) |
| 폐업 업체 표시 | `is_open` 필터(영업중만 지도 노출). 원천 `biz_status` 보존 |
| 좌표 오변환 | 원천 `src_x/src_y` 보존 + 변환 검증 절차(§4 메모) |
| 공공데이터 라이선스 | 출처 표기 의무 확인(데이터셋별 이용허락범위). 앱에 출처 고지 |

## 9. 엣지 케이스

- **좌표 없는 업체**: 원천에 좌표 누락 → `geom=null`, 지도에서 제외(주소만 보유). 목록형 보조 표시는 가능.
- **5174 변환 오차**: 일부 데이터셋은 보정계수/원점이 달라 수십 m 틀어질 수 있음 → 검증 후 SRID 조정.
- **애견카페 결과 5건 한계**: 네이버 지역검색 `display` 최대 5. **v1 은 단일 호출(상위 5곳)로 출발**, 밀집 지역 누락이 문제되면 격자 다중질의로 승급(§4-1). 위치 정렬 약함 → 5km 후필터.
- **애견카페 좌표계**: `mapx/mapy` 는 KATECH(TM128). CSV 의 EPSG:5174 와 **다른 변환**이니 혼동 금지.
- **애견카페 미노출 시간대**: 실시간 API 장애·쿼터 소진 시 애견카페만 빈다(CSV 시설은 정상). 토글로 분리돼 있어 다른 마커엔 영향 없음.
- **반려동물용품점**: 순수 소매(용품점)는 동물 인허가에 없을 수 있음 → `pet_sales`(동물판매업) 위주로만 커버됨을 UI 에 반영.
- **도심 마커 과밀**: `limit 500` + 네이버 지도 클러스터링(줌 레벨별 군집)으로 완화.
- **위치 권한 거부**: 0017 정책 재사용(기본 지역/수동 위치).
- **동기화 실패**: cron 실패 알림 + 다음 주기 재시도. 기존 데이터는 유지(부분 실패가 전체를 비우지 않게 upsert).
