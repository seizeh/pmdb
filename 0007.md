# 0007 · 전화 인증 전환

**파일**: `supabase/migrations/20260525120007_phone_auth.sql`
**카테고리**: 기능 변경
**의존성**: 0001~0006

## 목적

이메일 인증을 전화 OTP 로 교체합니다. email_verifications → phone_verifications 로 대체하고, users 테이블의 email 컬럼 제거 + phone 컬럼 추가. Solapi(구 CoolSMS) 연동을 전제로 6자리·5분 인증 코드 흐름.

## 핵심 변경

| 구분 | 내용 |
|---|---|
| 신규 테이블 | phone_verifications (phone, code, purpose=signup|password_reset, expires_at, is_used) |
| users 컬럼 | phone VARCHAR(20) 추가, phone_verified BOOLEAN 추가 |
| 삭제 | users.email, users.email_verified, users_lower_email_uq 인덱스, email_verifications 테이블 |
| 유일성 | users_phone_uq — 정규화된 번호에 대한 partial unique |
| RLS | phone_verifications 접근은 service_role 만 (정책 없음) |

## 전체 SQL

```sql
-- ============================================================================
-- PawMate · 0007 · 전화 인증 전환 (이메일 인증 제거)
-- ----------------------------------------------------------------------------
-- [결정] 전화번호를 주 신원수단으로. 이메일 컬럼/인증 제거.
--   · 가입/비번재설정: 전화 OTP(코드 6자리·5분) — email_verifications 패턴을 그대로 이식.
--   · 아이디(username) 찾기: 인증된 전화로 username 발송(스키마 변경 불필요).
--   · 신고 접수 확인(F-34): 이메일 → 인앱 알림(notifications) 또는 SMS 로 대체(서비스 레이어).
--
-- [SMS 연동 — Solapi(구 CoolSMS), SMS 8원/건]
--   서버(또는 Supabase Edge Function)에서:
--     1) POST https://api.solapi.com/messages/v4/send  (HMAC-SHA256 서명 헤더)
--        body: { message: { to, from(사전등록 발신번호), text: "[PawMate] 인증번호 123456" } }
--     2) 발송 직전 phone_verifications 에 (phone, code, purpose, expires_at=now()+5min) INSERT
--     3) 검증: 같은 phone+purpose 의 미사용·미만료 최신 code 일치 확인 → is_used=true
--   rate limit: 동일 번호 1분당 1회(서비스 레이어). 국내는 '발신번호 사전등록' 필요.
-- ============================================================================

-- ---------------------------------------------------------------------------
-- 1) phone_verifications (email_verifications 대체)
-- ---------------------------------------------------------------------------
create table if not exists public.phone_verifications (
  id          uuid primary key default gen_random_uuid(),
  phone       varchar(20)  not null,                 -- 정규화 저장(예: 01012345678 또는 +8210...)
  code        varchar(10)  not null,                 -- 6자리 코드
  purpose     varchar(20)  not null default 'signup'
                check (purpose in ('signup','password_reset')),
  expires_at  timestamptz  not null,                 -- 발급 후 5분
  is_used     boolean      not null default false,
  created_at  timestamptz  not null default now()
);
comment on table public.phone_verifications is '전화 인증/비번재설정 코드(6자리·5분). rate limit: 동일 번호 1분당 1회(서비스/엣지)';

create index if not exists phone_verifications_lookup_idx
  on public.phone_verifications (phone, purpose, created_at desc);
create index if not exists phone_verifications_expires_idx
  on public.phone_verifications (expires_at);

alter table public.phone_verifications enable row level security;
-- 정책 없음 → 클라이언트 접근 전면 차단. service_role 만 사용.
grant all on public.phone_verifications to service_role;

-- ---------------------------------------------------------------------------
-- 2) users : phone 추가 / email 제거
-- ---------------------------------------------------------------------------
alter table public.users add column if not exists phone          varchar(20);
alter table public.users add column if not exists phone_verified boolean not null default false;

-- 전화번호 유일성(식별자). 값이 있는 행만 유일 — 마이그레이션 안전.
-- (앱은 가입 시 항상 인증된 phone 을 설정. 커스텀 인증이라 service_role 로 INSERT.)
create unique index if not exists users_phone_uq
  on public.users (phone) where phone is not null;

-- 이메일 관련 제거 (인덱스 → 컬럼 순). 컬럼 GRANT(0006)에는 email 이 없어 영향 없음.
drop index if exists public.users_lower_email_uq;
alter table public.users drop column if exists email_verified;
alter table public.users drop column if exists email;

comment on column public.users.phone is '인증된 전화번호(신원/공동보호자 초대 매칭 키). PII → 컬럼 GRANT 미부여로 클라이언트 비공개';

-- ---------------------------------------------------------------------------
-- 3) email_verifications 제거 (RLS/인덱스 동반 삭제)
-- ---------------------------------------------------------------------------
drop table if exists public.email_verifications;
```

## 주의사항

- phone 은 PII 라 컬럼 GRANT 미부여 → 클라이언트 비공개, 서비스/엣지에서만 조회.
- 발신번호 사전등록(정보통신망법) 필요 — Solapi 콘솔에서 처리.
- 이후 0008 에서 초대 매칭(phone → invitee_user_id) 로직이 phone 을 활용.
