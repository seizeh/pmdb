# PawMate 변경 이력 (2026-06-08 ~ 2026-06-11)

최근 3일간의 변경을 **시간순**으로 정리한 문서입니다.
저장소는 둘로 나뉩니다.

- **pmdart** — Flutter 앱 (https://github.com/seizeh/pmdart)
- **pmdb** — Supabase DB/백엔드: 마이그레이션 + Edge Functions (https://github.com/seizeh/pmdb)

각 항목 끝의 `()` 안은 커밋 해시입니다. `app`=pmdart, `db`=pmdb.

## 2026-06-11 — 공동보호자·검색/조회수·초대·관리자 페이지

### 1) 공동보호자 지원자 관리
- 공동보호자도 다른 보호자의 게시글에서 **지원자 목록을 보고 대리 수락** 가능. 수락 시 약속의 보호자 측(`post_owner_id`)이 **작성자가 아닌 수락한 공동보호자**가 되고, 작성자에게는 `application_accepted_by_co` 알림 발송(작성자는 평가 불가). (app `b9e66db` / db `b9e4c11`)
  - DB: `is_post_manager` 헬퍼, `applications` RLS 확장, 수락 트리거 수정, 알림 타입 추가.

### 2) 반려동물 검색 + 펫 공개 프로필
- 사용자 검색에서 **닉네임 + 반려동물 이름** 병렬 검색 → "반려동물 / 보호자" 섹션 표시. 펫 결과 탭 시 사용자 프로필이 아닌 **read-only 펫 프로필**로 이동. (app `dd7018d`)

### 3) 게시글 조회수
- 게시글 진입 시 `post_views` 기록(트리거가 `view_count +1`) + 피드 카드·상세에 조회수 표시. (app `20fe11d`)
- 보정: **작성자 본인 조회 제외** + 중복 집계 리타이어를 **1시간 단위 버킷**으로 정정. (app `5a8d5a5`)
- 정리: 컴파일되지 않던 잔재 파일 `PostDetailScreen.java` 삭제. (app `b7b11d5`)

### 4) 공동보호자 초대 흐름
- **진행 중 약속 보호**: 진행 중(scheduled) 약속의 지원자는 그 펫의 보호자 초대를 **약속 완료 전까지 수락 차단**(초대는 pending 유지). (db `1d61dc3`)
- **기존 가입자 연결**: 초대 발송 시 전화번호가 이미 가입자면 `invitee_user_id` 즉시 연결(기존엔 가입 시점에만 연결). (db `2033378`)
- **받은 초대 화면**: 내 정보 > 내 활동에 "받은 보호자 초대"(대기 건수 배지) + 수락/거절 화면. (app `92d42e2`)
- **초대 알림**: 초대 시 수신자에게 `guardian_invite` 알림(발송 시 + 미가입자 가입 시), 알림 탭 시 초대함으로 이동. (app `b68c4c9` / db `bcdbf78`)

### 5) 관리자 페이지 (신규)
관리자 계정(`user_type='admin'`) 로그인 시 일반 화면 대신 **관리자 콘솔**로 진입. 모든 관리 기능은 `app.is_admin()`으로 게이트한 SECURITY DEFINER RPC로 구현(핵심 테이블의 컬럼 권한 제약 때문).

- **로그인 라우팅 + 대시보드**: 회원/정지회원/게시글/진행중 약속/미처리 신고 통계. (app `4af97a6` / db `6cd1798`)
- **회원 관리**: 닉네임·아이디·전화 검색, 정지·휴면·활성(본인·다른 관리자 보호) + 감사로그. (app `6a7429d` / db `c0ccc95`)
- **신고 처리**: 미처리/전체 조회, 검토중·처리완료·반려. (app `b2caa03` / db `fb231b6`, 파일명 정정 `8d3ea02`)
- **게시글/댓글 관리**: 게시글 숨김·삭제, 게시글별 댓글 숨김·복원(기존 감사 트리거와 중복 제거). (app `3ef44f5` / db `2b51751`)
- **문의 처리**: `admin_inquiry` 채팅방 목록 → 참여 후 기존 채팅으로 응대. (app `93456b1` / db `cae7eb6`)
- **문의 라벨 보존**: 관리자 참여 후에도 사용자에게는 상대가 계속 **'고객센터'**로 보이도록 `v_chat_rooms` 뷰 조정. (db `979c7b7`)
- **비밀번호 변경**(일반/관리자 공통): 현재 비번 확인 후 bcrypt 갱신. 내 정보 > 설정 / 관리자 홈 앱바. (app `b59b33c` / db `71409a5`)
- **신고 상세 + 감사 로그 + 전용 테마**: 신고 항목 탭 → 신고된 게시글/댓글/회원/채팅 **실제 내용 확인 + 즉시 조치**, 관리자 조치 이력(감사 로그) 화면, 관리자 전용 딥 슬레이트 테마 + 신원 헤더. (app `3c076f1` / db `59d1e35`)

### 참고 사항

- **DB 적용 방식**: 모든 DB 변경은 운영(Supabase)에 적용 후 동일 내용을 pmdb 마이그레이션으로 백필했습니다. 파일명 타임스탬프는 원격 마이그레이션 버전과 일치시킵니다.
- **검증**: DB 변경은 트랜잭션 픽스처 + 롤백 방식으로 동작/권한을 검증(운영 데이터 변경 0), 앱 변경은 `flutter analyze`로 확인했습니다.
- **인증/관리자 구조**: 커스텀 JWT(`app.uid()` = JWT `sub`), 관리자 기능은 `is_admin` 게이트 RPC. 자세한 결정은 저장소 내부 메모/주석 참고.
