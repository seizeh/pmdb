# 0011 · owner 탈퇴/정지 시 자동 승계

**파일**: `supabase/migrations/20260525120011_owner_succession.sql`
**카테고리**: 정책
**의존성**: 0008, 0009

## 목적

사용자가 active → inactive/suspended 로 전환될 때 owner 였던 펫을 자동 승계. 가장 먼저 들어온 co_guardian 에게 owner 이전, 후계 없으면 pet_status='deleted' 로 비활성화. 떠나는 사용자의 pet_guardians 행은 모두 정리.

## 핵심 변경

| 구분 | 내용 |
|---|---|
| 트리거 | app.tg_users_owner_succession (AFTER UPDATE OF status) |
| 승계 로직 | co_guardian 중 pet_guardians.created_at 오름차순 첫 번째에게 owner 이전 + pets.primary_guardian_id 갱신 |
| 고아 처리 | 후계 없으면 pet_status='deleted' 로 비활성화 (기록·이력은 보존) |
| 정리 | 떠나는 사용자를 강등 → 삭제 순으로 처리해 partial unique(one_owner) 충돌 회피 |

## 전체 SQL

```sql
-- ============================================================================
-- PawMate · 0011 · owner 탈퇴/정지 시 자동 승계
-- ----------------------------------------------------------------------------
-- 정책: 사용자가 active → inactive/suspended 로 전환될 때
--   · owner 였던 각 펫에 대해 co_guardian 중 "가장 먼저 들어온"(connection 순서)
--     사용자에게 owner 자동 승계.
--   · 승계할 co_guardian 이 없으면 그 펫은 pet_status='deleted' 로 비활성화.
--   · 떠나는 사용자의 pet_guardians 행은 모두 정리.
-- 트리거 순서·동시성 안전: 강등→승격→삭제 순으로 partial unique(one_owner) 충돌 회피.
-- ============================================================================

create or replace function app.tg_users_owner_succession()
returns trigger
language plpgsql
security definer
set search_path = ''
as $$
declare
  v_pet  uuid;
  v_heir uuid;
begin
  -- active → inactive/suspended 전이일 때만 동작
  if not (old.status = 'active' and new.status in ('inactive','suspended')) then
    return new;
  end if;

  -- 떠나는 사용자가 owner 인 펫 각각에 대해 처리
  for v_pet in
    select pet_id from public.pet_guardians
     where user_id = new.id and role = 'owner'
  loop
    -- 가장 먼저 들어온 co_guardian(연동 순서 우선) 후보 선정
    select user_id into v_heir
      from public.pet_guardians
     where pet_id = v_pet
       and role  = 'co_guardian'
       and user_id <> new.id
     order by created_at asc, id asc
     limit 1;

    if v_heir is not null then
      -- ① 떠나는 owner 를 먼저 co_guardian 으로 강등 (one_owner partial unique 충돌 방지)
      update public.pet_guardians
         set role = 'co_guardian'
       where pet_id = v_pet and user_id = new.id;
      -- ② 후계자 승격
      update public.pet_guardians
         set role = 'owner'
       where pet_id = v_pet and user_id = v_heir;
      -- ③ 소유자 포인터 갱신
      update public.pets
         set primary_guardian_id = v_heir
       where id = v_pet;
    else
      -- 후계 없음 → 펫 비활성화. (떠나는 사용자를 먼저 co_guardian 으로 강등해야
      -- 다음 단계의 DELETE 가 prevent-owner-self-remove 트리거에 막히지 않음.)
      update public.pet_guardians
         set role = 'co_guardian'
       where pet_id = v_pet and user_id = new.id;
      update public.pets
         set pet_status = 'deleted'
       where id = v_pet;
      -- primary_guardian_id 는 NOT NULL 이라 비활성 사용자 가리킴 그대로 둠
      -- (펫이 비활성이라 신규 활동에는 사용되지 않음 — 이력 보존용 참조만 유지)
    end if;
  end loop;

  -- 떠나는 사용자의 모든 보호자 행 정리 (owner 였던 펫은 위에서 강등됐고,
  -- 원래 co_guardian 이던 펫은 그대로 남아 있던 것을 일괄 제거)
  delete from public.pet_guardians where user_id = new.id;

  return new;
end;
$$;

create or replace trigger trg_users_owner_succession
  after update of status on public.users
  for each row execute function app.tg_users_owner_succession();
```

## 주의사항

- self-remove 차단(0009)을 우회하려고 강등 → 삭제 순서로 처리.
- suspended → active 복귀는 자동 되돌리지 않음 (이미 새 owner 자리 잡음 → 재조정 안 함).
- 관리자가 정지시키는 경우 app.uid() ≠ old.user_id 이므로 self-remove 가드 걸리지 않음.
